👁️ Nur Lesen
v26.0PTH Modding Team

Fast fertig — Lesen offen, Bearbeiten per Discord

Jeder mit dem Link kann alles lesen. Bearbeiten, kommentieren und neue Seiten anlegen geht nur nach Login mit Discord.

  1. Kostenlosen Account auf supabase.com erstellen, neues Projekt anlegen
  2. Das Basis-Skript sowie migration-v3.sql, migration-v4.sql, migration-v5.sql und migration-v6.sql im SQL Editor ausführen (Migrations-Dateien wurden dir separat bereitgestellt)
  3. Auf discord.com/developers/applications eine neue App erstellen, unter OAuth2 Client-ID und Client-Secret kopieren
  4. In Supabase unter Authentication → Providers → Discord aktivieren, Client-ID + Secret eintragen. Die dort gezeigte Callback-URL bei Discord unter OAuth2 → Redirects eintragen
  5. Optional für Bild-Uploads: unter Storage einen neuen öffentlichen Bucket namens wiki-images erstellen (siehe storage-policies.sql)
  6. URL + Publishable Key unten eintragen, hochladen

Hinweis: Jeder mit einem beliebigen Discord-Account kann sich einloggen und dann bearbeiten — nicht nur Mitglieder eures Servers. Eine Beschränkung auf euren Server bräuchte wieder eine zusätzliche Server-Funktion (hatten wir vorher schonmal skizziert).
